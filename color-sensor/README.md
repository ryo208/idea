# 色判別デバイス

ものに触れると色を判別して、スマホに飛ばしてくれる装置。

## iPhoneだけで足りるか?

**用途と精度次第。まずはiPhoneのみで検証するのがおすすめ。**

### 案A: iPhoneのみ(カメラで色判別)

- カメラで対象を写し、タップした点(または中央)のRGBを取得して色名に変換するアプリを作る
- ハードウェア不要、すぐ作れる、持ち歩くものが増えない

**弱点**: 環境光に大きく左右される
- 蛍光灯・夕日・影の下では同じ物でも違う色に見える(ホワイトバランスの問題)
- 対策: フラッシュ(トーチ)を常時点灯して至近距離で撮る/白い紙でキャリブレーションする
- それでも「だいたいの色」レベル。ペンキや布の厳密な色合わせには不十分

### 案B: 専用センサーデバイス + iPhone

「触れると判別」という体験に一番忠実な構成。

```
[カラーセンサー TCS34725] --I2C--> [ESP32] --BLE--> [iPhoneアプリ]
```

- **TCS34725**: 定番のRGBカラーセンサー。白色LED内蔵モジュールなら自前照明で測れるので環境光の影響をほぼ排除できる。モジュールで数百円〜
- **ESP32**(またはM5StickC/M5Atom): BLEでiPhoneに送信。M5系なら画面・バッテリー・ケース込みで工作が楽
- **iPhone側**: BLE受信アプリが必要
  - ⚠️ iOSのSafariはWeb Bluetooth非対応なので、Webアプリでは受け取れない
  - SwiftUI + CoreBluetoothでネイティブアプリを作るか、開発初期は既存のBLEユーティリティアプリ(LightBlue等)で値を確認する

**強み**: 接触測定+自前照明で照明条件に依存しない安定した測色
**弱点**: ハードの工作とネイティブアプリ開発の両方が必要

### 判断基準

| 用途 | 推奨 |
|---|---|
| 「これ何色?」がだいたい分かればいい(色覚サポート、日常用) | 案A(iPhoneのみ) |
| ペンキ・布・印刷物の色合わせなど精度が必要 | 案B(専用センサー) |
| 電子工作自体を楽しみたい | 案B |

### 進め方(推奨)

1. **Phase 1**: iPhoneのみのカメラ色判別アプリを作って使用感を検証
2. **Phase 2**: 精度に不満が出たら TCS34725 + ESP32 のセンサーデバイスを追加し、アプリにBLE受信を実装

## 想定される用途

1. **色覚サポート**(本命): 色の見分けが難しい人の日常サポート。服選び・買い物・熟れた果物の判別など。日本では男性の約5%(20人に1人)が色覚多様性
2. **視覚障害サポート**: 色名の読み上げで服の管理などを支援
3. **服のコーディネート・洗濯仕分け**: 「この2つ、同じ紺?黒?」問題の解決
4. **DIY・インテリア**: 壁や家具の色をカラーコードでメモして、買い物時に照合
5. **デザインの色採取**: 現実の物からカラーコードを取得するリアル版スポイトツール
6. **知育**: 子供と色当て遊び(読み上げ機能と相性が良い)

## メモ

- 市販の類似品: Nix Mini、Datacolor ColorReader(接触式測色計、1〜3万円)→ 案Bはこれの自作版
- 色名変換: RGB → HSV変換して色相で分類 + 代表色との距離計算(CIEDE2000ならより人間の感覚に近い)
